# PRD: 모임 이벤트 관리 웹 MVP

> **버전**: v1.0  
> **작성일**: 2026-05-06  
> **상태**: 기획 완료

---

## 1. 배경 및 목적

### 문제 정의

수영, 헬스, 친구 모임 등 소규모 정기 모임을 주최할 때 주최자는 다음 작업을 **각기 다른 도구**로 처리해야 한다:

| 업무        | 현재 방식                | 문제점                             |
| ----------- | ------------------------ | ---------------------------------- |
| 모임 공지   | 카카오톡 단톡방          | 공지가 메시지에 묻혀 찾기 어려움   |
| 참여자 관리 | 스프레드시트             | 수동 업데이트, 실수 발생           |
| 카풀 조율   | 단톡방 댓글              | 혼선, 중복 신청                    |
| 정산        | 토스/카카오페이 + 계산기 | 계산 과정이 불투명, 반복 설명 필요 |

→ 이 모든 업무를 **하나의 링크**로 처리할 수 있는 도구가 필요하다.

### 목표

주최자가 이벤트 링크 하나를 공유하면, 참여자들이 별도 앱 설치나 회원가입 없이 참여 신청·카풀·정산을 처리할 수 있는 **경량 웹앱 MVP** 제공.

---

## 2. 타겟 사용자

### 주최자 (Organizer)

- **누구**: 정기 모임을 주도하는 1인
- **특징**: 로그인 필요, 이벤트 생성·관리 권한 보유
- **Pain Point**: 참여자 취합 → 카풀 매칭 → 정산 계산을 반복적으로 직접 처리

### 참여자 (Participant)

- **누구**: 주최자로부터 링크를 받은 모임 구성원
- **특징**: 로그인 불필요, 링크로 바로 접근
- **Pain Point**: 복잡한 가입 절차 없이 빠르게 참여 의사를 전달하고 싶음

### 규모

- 모임당 참여자 **5~20명** 소규모 중심

## 3. 핵심 기능 (MVP Scope)

### 3-1. 이벤트 생성 및 공지

**주최자** 기능:

- 이벤트 생성: 제목, 날짜/시간, 장소, 설명, 최대 인원
- 이벤트 수정 및 상태 관리 (활성 / 마감)
- **공유 링크 자동 생성**: `/events/{public_token}` 형식의 고유 URL

**참여자** 기능:

- 링크 접속 시 이벤트 정보 즉시 확인
- 현재 참여자 수 / 최대 인원 표시

---

### 3-2. 참여자 관리

**참여자** 기능:

- 로그인 없이 참여 신청 (입력 항목: 이름*, 전화번호*, 메모)
- 신청 후 목록에 이름 노출

**주최자** 기능:

- 참여자 목록 조회 (이름, 연락처, 메모, 신청 시간)
- 참여 상태 변경: `대기중 → 확정 / 취소`
- 확정 인원 기준으로 정산 자동 계산

---

### 3-3. 카풀 조율

**참여자** 기능:

- **드라이버 등록**: 출발지, 가능 인원, 메모 입력
- **탑승 신청**: 드라이버 카드에서 자신의 이름을 선택해 신청
- **탑승 취소**: 신청 철회 가능

**표시 정보**:

- 드라이버 카드: 출발지, 가능 좌석 수, 현재 탑승자 목록
- 남은 자리 실시간 업데이트

---

### 3-4. 정산

**주최자** 기능:

- 비용 항목 추가/삭제: 항목명, 금액, 낸 사람
- 실시간 합계 계산

**참여자 / 주최자 공통**:

- 총 비용 합계 표시
- 확정 참여자 수 기준 **1/N 균등 분담** 자동 계산
- "1인당 OOO원" 명확하게 표시

---

### 3-5. 알림

- 참여자가 신청할 때 → 주최자 이메일로 실시간 알림
- 알림 내용: 이벤트명, 신청자 이름, 전화번호

---

## 4. 사용자 플로우

### 주최자 플로우

```
로그인
  → 이벤트 대시보드 (/protected/events)
  → 새 이벤트 생성 (/protected/events/new)
  → 이벤트 관리 페이지 (/protected/events/[id])
    ├── 공유 링크 복사
    ├── 참여자 확정/취소
    └── 비용 항목 입력
```

### 참여자 플로우

```
링크 클릭
  → 공개 이벤트 페이지 (/events/[token])
    ├── 이벤트 정보 확인
    └── 참여 신청 (이름, 전화번호, 메모)
  → 카풀 페이지 (/events/[token]/carpool)
    ├── 드라이버 등록 또는
    └── 탑승 신청
  → 정산 페이지 (/events/[token]/settlement)
    └── 1/N 금액 확인
```

## 5. 라우트 구조

```
app/
├── page.tsx                          # 랜딩 페이지 (이벤트 만들기 CTA)
├── events/
│   └── [token]/
│       ├── page.tsx                  # 공개 이벤트 페이지 + 참여 신청
│       ├── actions.ts                # joinEvent, registerDriver, requestRide
│       ├── carpool/
│       │   └── page.tsx              # 카풀 현황
│       └── settlement/
│           └── page.tsx              # 정산 현황
└── protected/
    └── events/
        ├── page.tsx                  # 내 이벤트 대시보드
        ├── new/
        │   └── page.tsx              # 이벤트 생성 폼
        └── [id]/
            ├── page.tsx              # 이벤트 관리
            └── actions.ts            # createEvent, updateStatus, addExpense
```

## 6. 데이터 모델

### events

| 컬럼             | 타입        | 설명                 |
| ---------------- | ----------- | -------------------- |
| id               | uuid PK     |                      |
| organizer_id     | uuid FK     | auth.users           |
| title            | text        | 이벤트 제목          |
| description      | text        | 상세 설명            |
| event_date       | timestamptz | 일시                 |
| location         | text        | 장소                 |
| max_participants | int         | 최대 인원 (nullable) |
| public_token     | text UNIQUE | 공유 링크용 토큰     |
| status           | text        | `active` \| `closed` |

### participants

| 컬럼      | 타입        | 설명                                    |
| --------- | ----------- | --------------------------------------- |

| name      | text        | 참여자 이름                             |

## 8. 비기능 요구사항

- **반응형**: 모바일 우선 (참여자 대부분 스마트폰으로 접근)
- **링크 접근성**: 로그인 없이 즉시 참여 가능
- **보안**: Supabase RLS로 이벤트 데이터 격리 (주최자만 수정 가능)
- **성능**: Server Components 기반 빠른 초기 로딩

## 9. MVP 범위 외 (v2 고려사항)

| 기능                      | 이유                    |
| ------------------------- | ----------------------- |
| 실시간 채팅               | MVP 복잡도 증가         |
| 커스텀 정산 (불균등 분담) | 1/N으로 MVP 커버 가능   |
| 반복 이벤트 (정기 모임)   | 단순 생성 반복으로 대체 |
| 푸시 알림                 | 이메일로 MVP 커버       |
| 투표/설문 (날짜 조율)     | 별도 기능으로 추후 추가 |
| 모임 멤버십/그룹 관리     | 이벤트 단위로 MVP 충분  |
