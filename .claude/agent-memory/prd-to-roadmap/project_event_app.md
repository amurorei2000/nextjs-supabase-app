모임 이벤트 관리 웹앱 MVP를 개발 중이다. PRD v1.0 기반, 로드맵 v1.0 작성 완료(2026-05-06). Phase 0(DB) → Phase 1(이벤트 생성+참여신청) → Phase 2(카풀+정산+이메일+UX) 3단계, 총 7주 계획.

핵심 설계 결정:

- 비인증 참여자 식별: 이름+전화번호 조합으로 participant 레코드 매칭 (ADR-01)
- Resend 알림 실패는 참여 신청 롤백 없음 (ADR-02)
- public_token: crypto.randomUUID() 서버 생성, DB UNIQUE 제약

**Why:** 소규모 정기 모임(5~20명) 주최자의 공지·카풀·정산 업무를 단일 링크로 통합하는 것이 핵심 가치 제안. 참여자 진입 마찰 최소화(로그인 불필요)가 최우선 비기능 요구사항이다.

**How to apply:** 구현 제안 시 "참여자는 비인증"이라는 전제를 항상 고려한다. RLS 설계, Server Action 권한 체크, UI에서 주최자 전용 기능 분기 시 이 전제를 기준으로 판단한다.
